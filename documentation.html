NEST Entrance App

A secure, full-stack online entrance examination platform with real-time timer, role-based access, and an admin control panel.

📋 Overview

NEST Entrance App (internally called The Scholarly Monolith) is a browser-based entrance examination system. Administrators build a question bank, invite candidates by email, and control exam availability via a master toggle. Candidates log in with auto-generated credentials, take a strictly-timed multiple-choice exam, and submit answers — all within a secured, role-restricted session.

👤 User Roles

🛡️ Administrator

Accesses the system at /admin/login. Full control over questions, candidate management, exam settings, and system status. Default credentials: admin / adminpassword (set on first database initialisation).

🎓 Student / Candidate

Accesses the system at / (root login). Uses email as username and the auto-generated password sent by the admin. Can only enter the exam when the admin has enabled it. Answer submission is final.

✨ Features

Authentication & Session Management

  • Separate login portals for students and admins
  • Passwords hashed with bcryptjs (10 salt rounds)
  • Sessions managed via JWT tokens with 12-hour expiry
  • Protected routes: unauthenticated users redirected to login
  • Role verification on every protected API request

Student Exam Interface

⏱ Countdown Timer

Live mm:ss countdown. Turns red and pulses when fewer than 5 minutes remain. Automatically submits the exam when time reaches zero.

📊 Progress Tracking

Animated progress bar showing the ratio of answered to total questions. Instantly updates on every selection.

🗺 Question Navigation

Previous / Next buttons for sequential navigation. Current question number and total count always visible.

✅ Answer Selection

Four radio-button options per question. Previously selected answers are remembered when the student navigates back.

📤 Submission Screen

Confirmation overlay after submission. "Concluded Session" button ends the session cleanly.

🔒 Exam Gate

If admin has disabled the exam, students are shown an "Exam not available" message and cannot proceed.

Admin Dashboard

The admin panel is organised into four tabs:

Tab 1 Dashboard Overview
  • Total candidate count
  • Total question count
  • System integrity percentage
  • Exam analytics bar chart
Tab 2 Candidate Access Management
  • Invite a candidate by entering their email address
  • System auto-generates a secure random password
  • Credentials dispatched via Nodemailer (Ethereal test transport)
  • Invited candidates list with status badges
Tab 3 Question Bank ("The Question Monolith")
  • Create multiple-choice questions: question text + 4 option texts + mark correct answer
  • View full question list with correct answer highlighted in green
  • Delete individual questions
  • Questions numbered sequentially
Tab 4 Exam Global Rules
  • Master enable / disable toggle for the exam
  • Setting persists to the database — survives server restarts
  • When disabled: students receive a 403 response and a friendly UI message

Email Notifications

ℹ️
The email system uses Ethereal (a fake SMTP service). Emails are not actually delivered to inboxes — instead a preview URL is logged to the server console. Replace with a real SMTP provider for production use.

🔌 API Reference

All endpoints are prefixed with /api. Protected routes require the header Authorization: Bearer <jwt_token>.

Public Endpoints

POST
/api/auth/login Authenticate a user. Body: { username, password }. Returns { token, role }.

Admin-Only Endpoints auth + admin role required

GET
/api/questionsRetrieve all questions (includes correct answers).
POST
/api/questionsCreate a question. Body: { text, options: string[4], correct_answer }.
DELETE
/api/questions/:idDelete a question by ID.
GET
/api/exam-settingsGet current exam settings (enabled flag, duration, window times).
PUT
/api/exam-settingsUpdate exam settings. Body: { is_enabled, duration_minutes, start_time, end_time }.
GET
/api/studentsList all student accounts.
POST
/api/students/inviteInvite a student. Body: { email }. Creates account, sends mock email with credentials.

Student-Only Endpoints auth + student role required

GET
/api/examReturns exam questions (correct answers stripped) and settings. Returns 403 if exam is disabled.

🔄 User Workflow

Admin Setup Flow

Log in as AdminNavigate to /admin/login and sign in with admin / adminpassword.
Build Question BankOpen the "Question Bank" tab. Add questions: provide text, four options, and mark the correct one.
Invite CandidatesOpen "Candidate Access" tab. Enter each candidate's email. System creates an account and sends (mock) credentials.
Enable the ExamOpen "Exam Rules" tab. Flip the master toggle to Enabled. Students can now access the exam.
MonitorDashboard tab shows live candidate and question counts.

Candidate Exam Flow

Receive CredentialsCandidate receives email (or is told verbally in dev mode) their username (email) and auto-generated password.
Log InNavigate to root /. Enter credentials on the student login page.
Take ExamTimer starts immediately. Navigate questions with Prev / Next. Select one answer per question.
SubmitClick "Submit" when finished — or wait for auto-submit when the timer expires.
ConcludeConfirmation screen appears. Click "Concluded Session" to end.

🚀 Setup & Running

  • Node.js ≥ 18
  • npm ≥ 9

npm install          # first time only
node server.js       # starts on http://localhost:3001

On first run the SQLite database is created, tables are seeded, and a default admin account is added automatically.

🔐 Security

Password Hashing

All passwords stored as bcrypt hashes (10 salt rounds). Plain-text passwords are never persisted.

JWT Authentication

Stateless token-based sessions. Tokens expire after 12 hours. Secret key validated on every protected request.

Role-Based Access Control

Admin and student routes are independently guarded. A student token cannot access admin endpoints.

Answer Hiding

The student GET /api/exam endpoint strips correct_answer fields before returning questions.

Exam Gate

Server-side enforcement: exam access returns HTTP 403 when is_enabled = false, regardless of client state.

CORS

Express CORS middleware restricts which origins can call the API. Configure allowed origins for production.

🚨
Before deploying to production: change the hardcoded JWT secret in server.js, change the default admin password, switch sync({ force: true }) to migrations, and configure a real SMTP provider.
